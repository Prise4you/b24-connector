# b24-connector

Python-коннектор Bitrix24 → NotebookLM/BitrixGPT. Выгружает данные проекта по REST API Битрикс24 в Markdown-документы и заливает их в NotebookLM.

## Что делает

- Задачи + комментарии (`tasks.task.get`, `task.commentitem.getlist`) — полный набор полей: постановщик, ответственный, соисполнители, теги, статус.
- Чат группы (`im.dialog.messages.get`).
- Файлы диска (`disk.folder.getchildren`) — метаданные, не содержимое.
- Открытые линии (диалоги с клиентами) — **только точечно, по явному выбору пользователя** через `open_lines_data` в конфиге. НЕ выгружать все диалоги подряд — так было переделано умышленно после фидбека (лишние ПДн клиентов).
- CRM (компании/контакты/сделки/смарт-процессы) — **только по явно указанным `company_ids`**, не «выгрузить всё». См. `sources.crm` в конфиге.

## Архитектура

Git-репозиторий с GitHub-ремоутом (`Prise4you/b24-connector`). Синхронизация идёт через GitHub Actions по `repository_dispatch` — не требует включённого Mac пользователя. Триггер приходит из `../admin/api/webhook.php` при событиях в Б24.

Коннектор читает `config.json` (либо локально, либо через `--config-url`) — этот конфиг редактируется через панель `../admin`, не напрямую.

Локально живёт рядом со своей парной панелью в общей папке: `~/b24-notebooklm/connector` (эта папка, git) + `~/b24-notebooklm/admin` (панель, без git). Общий обзор — [[../CLAUDE.md]] на уровне родителя.

Локальный launchd-опрос (`launchd/com.anit.b24kb.poll.plist`, каждые 10 мин, `--poll`) — легаси с этапа до перехода на GitHub Actions, **отключён 2026-07-03** (`launchctl unload` + удалён из `~/Library/LaunchAgents/`), т.к. дублировал GitHub Actions и вызывал двойные прогоны синхронизации. Файлы `launchd/install.sh` и `launchd/com.anit.b24kb.poll.plist` оставлены в репозитории на случай, если понадобится вернуть (например, если GitHub Actions станет недоступен) — но по умолчанию НЕ включать без явной причины.

## Ключевые файлы

- `connector.py` — main, extract/build/load, вся логика выгрузки
- `config.example.json` — актуальная схема конфига
- `security.py` — фильтр секретов (токены, телефоны, СНИЛС), обязателен на всех путях выгрузки в NotebookLM
- `load_notebooklm.py` — загрузка документов в NotebookLM

## Жёсткое правило (152-ФЗ)

Открытые линии и CRM — это персональные данные клиентов. Наружу (в NotebookLM, США) идут только с осознанным согласием и точечным выбором пользователя. Массовая выгрузка запрещена архитектурно — не возвращать это поведение.

## Связанный проект

`../admin` (`~/b24-notebooklm/admin`) — PHP-панель (НЕ git-репозиторий, деплоится вручную на Beget). Пишет `config.json`, который читает этот коннектор.
